This article will outline how to use pipes in Angular to render icons based on the string you pass to a DOM Element.

This approach is useful for reuse and also, creates a richer UI when displaying things like lists and keywords, much like the example at the top of this article under Keywords:

So to get started, using the Angular CLI create a pipe using the command: 

  
    ng g p images

This will create a pipe with a test spec wherever you have chosen to set it up i.e. in a root dir, or utils, wherever.

Inside the new pipe file you will already be extending the use of the PipeTransform interface, so just ensure that your transform method returns a string and remove all unnecessary params, leaving a value:string option as below:

  
    @Pipe({
      name: 'images'
    })
    export class ImagesPipe implements PipeTransform {

      transform(imageUrl: string): string {
        ...
      }
    }

We can make use of a couple of techniques here to make things are nice and easy to scale and maintain, so first, add a switch case to the transform method as below and pass a boolean value of true as an argument:

  
    @Pipe({
      name: 'images'
    })
    export class ImagesPipe implements PipeTransform {

      transform(imageUrl: string): string {
        switch (true) {
          ...
        }
      }
    }

In this case I am rendering specific icons for specific technologies, so using two Enums, I have one for a technology type and one for the image it accompanies: 

  
    export enum TechnologyEnum {
      Angular = 'Angular',
    }

And a LogoEnum as so: 

  
    export enum LogoEnum {
      Angular = '../../assets/img/angular_logo.svg'
    }
  


  
    @Pipe({
      name: 'images'
    })
    export class ImagesPipe implements PipeTransform {

      transform(imageUrl: string): string {
        switch (true) {
          case imageUrl.includes(TechnologyEnum.Angular): return LogoEnum.Angular;
        }
      }
    }

So breaking this down and it's pretty simple, the switch will check for the imageUrl, then return a relevant file path, if the string uses the term. The imageUrl option passed to the transform method will be the technology type and if it includes say, Angular, it will bring back the icon in the assets dir for the angular icon.

This then can be scaled with other technologies and icons such as Typescript, where the pipe can be extended further as so:

  
    @Pipe({
      name: 'images'
    })
    export class ImagesPipe implements PipeTransform {

      transform(imageUrl: string): string {
        switch (true) {
          case imageUrl.includes(TechnologyEnum.Angular): return LogoEnum.Angular;
          case value.includes(TechnologyEnum.Typescript): return LogoEnum.Typescript;
        }
      }
    }

With the path for the typescript icon added to the LogoEnum as so: 

  
    export enum LogoEnum {
      Angular = '../../assets/img/angular_logo.svg',
      Typescript = '../../assets/img/typescript_logo.svg',
    }
