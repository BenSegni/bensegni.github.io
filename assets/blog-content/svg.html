This article explores using SVG code instead of HTML templates for building certain components. This approach can be especially useful when your component needs icons or images with specific requirements.

Here's why using SVG code can be beneficial:

  • Simplified code: Instead of using data binding with img elements, you can simply embed the SVG code directly, reducing the complexity of your code.
  • Dynamic content: You can apply business logic directly to the SVG code, allowing you to modify its content based on your application's needs.
  • Smooth animations: SVGs excel at animations, making them ideal for dynamic and engaging components compared to static images.

Let's take a look at the standard compilation of an Angular component:

  
    @Component({
      selector: 'app-blog',
      templateUrl: './blog.component.html'
    })
notice the templateUrl option inside the component decorator.

Using an svg instead of HTML 

  
    @Component({
      selector: 'app-github-icon',
      templateUrl: './icon.svg'
    })

As you can see, we have simply switched out the html with an svg file. The icon.svg file looks like this:

We can now use data binding techniques to this code, we can apply animations using angular animations, or structural directives to the svg code as we see fit.
