This article outlines how to use Typescript & Angular to create resuable animations. The approach will create opportunities the implementation of animations which can then be applied to other areas of the app.

The article will outline how to make use of a simple fade animation yet, this process can applied to many other edge cases which may be more complex in nature.

The first step will be to create a simple exportable function called FadeIn() - the return declaration will be of type AnimationTriggerMetadata imported from @angular/animations. For now, the function can just return a trigger method, imported again from @angular/animations. 

  
    import  { AnimationTriggerMetadata }  from "@angular/animations";

    export function FadeIn(): AnimationTriggerMetadata {
      return trigger();
    }
Next, add the necessary methods into the function to create the fade animation. 
  
    import  { AnimationTriggerMetadata, animate, style, transition, trigger }  from "@angular/animations";

    export function FadeIn(): AnimationTriggerMetadata {
      return trigger('fade', [
        transition('* => *', [
        style({ opacity: 0 }),
          animate(1000, style({ opacity: 1 }))
        ])
      ]);
    }
breaking the methods down inside the function:
  1. trigger - creates a name for the animation trigger which can be bound to the html.
  2. transition - the method controlling from & to states.
  3. style - the initial style state at the start of the animation.
  4. animation - the animation step which considers time & style state change.
These are annotated below: 
  
    import { AnimationTriggerMetadata, animate, style, transition, trigger } from "@angular/animations";

    export function FadeIn(): AnimationTriggerMetadata {
      // we name the trigger 'fade'.
      return trigger('fade', [
      // from and two states are the same, as they will switch between true & false.
        transition('* => *', [
        // initial styling makes the element not visible.
        style({ opacity: 0 }),
        // after 1 second, the element appears.
          animate(1000, style({ opacity: 1 }))
        ])
      ]);
    }

Once the animation has been completed, the function is now ready to be shared across the application. The implementation is very easy. Below is an example: 

  
    import { Component } from '@angular/core';
    import { FadeIn } from '../utils/animations/fade.animation';

    @Component({
        selector: 'app-some-component',
        templateUrl: './some-component.component.html',
        animations: [FadeIn()],
    })
    export class SomeComponent {
      ...
The animation in this example is found in a utils directory, but this can be set anywhere which best suits. As displayed, the only requirement after the function has been exported, is to import it into the animations property of the component decorator. Once this has been done, the trigger name can be bound to the html element which has been marked for animation.

The animation trigger name is bound to the div element and passing the show boolean into the animation i.e. true or false, the transition will show or hide the element using a fade animation.

This animation can be applied to any component across the application where it is considered suitable.
